# Orbital Chaos

**Orbital Chaos** — это симулятор 2D N-Body гравитации, разработанный на Java. Проект предназначен для реалистичной симуляции взаимодействия множества небесных тел с упором на производительность и возможность работы в многопоточной среде.

## Архитектура
Архитектура проекта строго разделяет математическую логику и визуализацию. Движок поддерживает **headless-режим** (режим без графического интерфейса), что подготавливает его к будущей интеграции с фреймворками вроде LibGDX или Java2D. Графическая подсистема может быть добавлена поверх текущего математического ядра без необходимости изменять его код.

## Интеграция физики
Для расчета физики движения используется метод интеграции **Velocity Verlet** (или RK4).
**Важно:** Использование простой интеграции Эйлера строго запрещено, так как это приводит к деградации орбит (orbit degradation) из-за накопления численной ошибки.

В текущей реализации движок использует метод `update` в классе `PhysicsEngine` для пересчета позиций, ускорений и скоростей тел.

## Оптимизации и управление памятью
Математические векторные операции (`Vector2D`) оптимизированы с учетом особенностей сборщика мусора (Garbage Collection).
Вместо создания новых объектов на каждый кадр симуляции используются **мутации векторов на месте (in-place mutations)** с помощью методов `set`, `add`, `sub`, и `scl`. Это позволяет избежать лишних аллокаций памяти и минимизировать задержки, вызываемые сборщиком мусора.

## Поддержка предсказаний ИИ
Движок симуляции поддерживает глубокое копирование (Deep Copy) состояния системы.
Метод `PhysicsEngine.deepCopyState` и copy-конструктор в `CelestialBody` позволяют ИИ-модулям или другим подсистемам создавать независимые копии текущего состояния игры. Эти копии могут быть использованы для симуляции будущего (предсказания орбит) в отдельных потоках без прерывания или нарушения текущего рендеринга и основного потока обновления.

## Запуск и использование
Так как в текущем виде движок представляет собой библиотеку/ядро (математическую модель), он не содержит встроенной графической оболочки или точки входа с `main` методом по умолчанию.

Для использования ядра в вашем проекте:
1. Создайте экземпляры `CelestialBody`, задав им массу, радиус, начальную позицию и скорость.
2. Сформируйте список тел `List<CelestialBody>`.
3. Инициализируйте движок `PhysicsEngine engine = new PhysicsEngine(G, softening);`, указав гравитационную постоянную и параметр сглаживания (для предотвращения сингулярностей при близких столкновениях).
4. Вызывайте `engine.update(deltaTime, bodies);` в вашем основном игровом цикле.

### Пример кода:
```java
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        // Создаем тела
        Vector2D pos1 = new Vector2D(0, 0);
        Vector2D vel1 = new Vector2D(0, 0);
        CelestialBody sun = new CelestialBody(1000, 50, pos1, vel1);

        Vector2D pos2 = new Vector2D(200, 0);
        Vector2D vel2 = new Vector2D(0, 50);
        CelestialBody planet = new CelestialBody(10, 5, pos2, vel2);

        List<CelestialBody> bodies = new ArrayList<>();
        bodies.add(sun);
        bodies.add(planet);

        // Инициализируем физический движок
        // G = 6.67430e-11 (или любая другая константа для вашей симуляции)
        // softening = 1.0
        PhysicsEngine engine = new PhysicsEngine(6.67430e-11, 1.0);

        // Симуляция 100 шагов
        double deltaTime = 0.016; // ~60 FPS
        for (int i = 0; i < 100; i++) {
            engine.update(deltaTime, bodies);
            System.out.println("Planet position: (" + planet.position.x + ", " + planet.position.y + ")");
        }
    }
}
```
